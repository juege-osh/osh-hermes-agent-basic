# Hermes Agent 到底是什么

## 一句话理解

Hermes Agent 是 Nous Research 开源的一套智能代理系统。它不只是一个聊天界面，而是一个把大模型、工具调用、终端执行、记忆、技能、消息平台接入和自动化任务组合起来的 agent 运行框架。

## 如果你是小白，可以这样理解

你可以把它拆成 5 层：

1. 最底层是大模型。
2. 上面一层是 provider 配置。
3. 再上面是 agent 逻辑。
4. 再上面是工具系统。
5. 最上面是你实际使用它的入口，比如 CLI、Telegram、Discord、cron、编辑器集成。

所以 Hermes 不是“模型本身”，而是“把模型变成可工作的代理”的那一层系统。

## Hermes 和普通聊天网页的差别

普通聊天网页通常只做这几件事：

- 发消息
- 收回答
- 保留短会话

Hermes 额外能做的事情包括：

- 调用终端
- 读写文件
- 使用多种工具集
- 管理技能
- 连接 MCP server
- 在不同平台继续同一个代理
- 做计划任务
- 存会话和记忆

## Hermes 适合谁

- 想在本地或服务器跑长期可用 AI 助手的人
- 想做带工具调用的代理工作流的人
- 想从 CLI 开始，再扩到 Telegram/Discord/Slack 的人
- 想要自己掌控模型提供商、工具权限和部署环境的人

## Hermes 不适合谁

- 完全不想接触命令行的人
- 想一步到位变成全自动商业系统的人
- 没有 provider key、也不准备配环境的人

## 官方能力范围

根据官方文档和 README，Hermes Agent 重点覆盖：

- CLI 交互
- 多 provider 模型切换
- tools 和 toolsets
- skills
- memory
- MCP
- cron 调度
- 消息平台网关
- 多种 terminal backend

## 你应该先建立的几个正确认知

### 1. Agent 不等于模型

模型是大脑，agent 是工作系统。没有合适的工具、权限和配置，再强的模型也很难稳定执行复杂任务。

### 2. 会回答问题不等于会稳定执行任务

很多新手第一次跑通后会觉得“已经好了”，但真正长期稳定运行，靠的是：

- provider 配置
- 工具可用性
- 权限边界
- 终端后端
- 会话持久化
- 失败重试与排错策略

### 3. 部署成功不等于生产可用

本地能用只是第一关。真正上 VPS、Docker、消息平台和计划任务后，问题才会开始复杂起来。

## 先记住这些高频名词

- `provider`：模型提供商
- `model`：实际调用的模型名
- `tool`：单个工具能力
- `toolset`：工具集合
- `skill`：按需加载的任务说明和流程
- `MCP`：Model Context Protocol，用来接外部能力
- `gateway`：把 Hermes 接到消息平台的网关
- `backend`：终端执行后端，比如 local、docker、ssh、modal

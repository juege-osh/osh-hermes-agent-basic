# 第一次用 CLI 真正跑起来

## 为什么先学 CLI

因为 CLI 是 Hermes 最直接、最容易排错的入口。

如果你连 CLI 都还没稳定跑通，就先不要着急上：

- Telegram
- Discord
- Slack
- 定时任务
- Docker 长期服务

## 首次进入前要确认什么

### 必须确认

- `hermes` 命令存在
- 至少一个 provider key 已配置
- 你知道自己准备先用哪个 model

## 进入 CLI

```bash
hermes
```

如果你想先单独挑模型：

```bash
hermes model
```

## 第一次进入后，你会碰到什么

通常你会先面对这些场景：

- 需要选择 provider
- 需要选择 model
- 需要确认工具配置
- 可能需要补充 `.env` 或 config

## 第一次提问建议

不要一上来就问超复杂任务，先做最小验证。

建议这样问：

```text
请简单介绍一下你当前可用的核心能力，并告诉我当前会话是否已经正常工作。
```

## 然后测试工具感知

再问：

```text
如果我让你查看当前目录结构，你会通过什么方式完成？
```

这一步不是一定要它立刻执行，而是先观察它是否理解工具调用语境。

## CLI 常见高频命令

```bash
hermes
hermes model
hermes tools
hermes setup
hermes doctor
hermes update
```

## 会话相关理解

Hermes 的好处之一是它不是每次都完全失忆。

你要理解几个常见动作：

- 新开会话
- 继续旧会话
- 压缩上下文
- 查看用量

## 为什么会话有时“像失忆”

常见原因：

1. 你换了 profile
2. 你换了平台
3. 会话没保存好
4. 发生了上下文压缩
5. 你以为 memory 和 session history 是一回事

它们不是一回事。

## Slash 命令为什么要学

CLI 不只是纯聊天，还支持很多 slash 命令。它们是高频提效入口。

比如你经常会用到：

- `/new`
- `/reset`
- `/model`
- `/usage`
- `/compress`
- `/retry`
- `/undo`

## 新手操作建议

第一次一小时内，只做这 5 件事：

1. 能进 CLI
2. 能切 model
3. 能发一条普通消息
4. 能看一次工具配置
5. 能理解会话并继续一次历史对话

## 常见错觉

### 错觉 1：它不报错就是一切正常

不对。有时 provider 配得不完全对，也可能只是“勉强有回话”。

### 错觉 2：会话能延续就等于 memory 生效

不对。session 和 memory 是不同层面的能力。

### 错觉 3：CLI 能用，消息平台就一定能用

不对。消息平台多了 gateway、授权、回调、平台限制等问题。

## 建议你做的第一个小练习

让 Hermes 帮你做一个只读任务，例如：

```text
请帮我总结当前项目目录结构应该如何组织成一套新手教程仓库，只先给我结构建议，不要修改任何文件。
```

这样你可以先观察它的回答质量和操作边界。

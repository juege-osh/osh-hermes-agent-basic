# 引导式实操

## 为什么要做实操卷

因为“看懂”和“做出来”中间差很远。

这一卷专门把学习拆成渐进练习。

## 练习 1：完成安装闭环

目标：

- 能安装 Hermes
- 能验证命令
- 能看懂 doctor 输出

动作：

1. 跑安装脚本
2. 跑验证脚本
3. 记录你的结果

## 练习 2：完成 provider 闭环

目标：

- 能配置至少一个 provider
- 能实际发出第一条测试消息

动作：

1. 填 `.env`
2. `source .env`
3. 跑 `hermes setup`
4. 跑 `hermes`

## 练习 3：完成 CLI 闭环

目标：

- 能新建会话
- 能继续会话
- 能切换 model

## 练习 4：理解工具闭环

目标：

- 知道 `hermes tools` 是干什么的
- 知道 backend 是什么
- 知道什么时候该换 Docker

## 练习 5：做一次安全思考

写下你的答案：

1. 你准备让 Hermes 跑在本机还是 VPS
2. 你是否允许它直接执行本机命令
3. 你是否计划接入消息平台
4. 如果多人都能和它说话，你准备怎么控权

## 练习 6：模拟一次排错

假设现象：

- CLI 能正常回答
- Telegram 没有回复

你应该优先检查什么？

标准思路：

1. token
2. gateway
3. allowlist
4. 平台状态

## 练习 7：规划你的部署路线

请从下面三种里选一条当前最适合你的：

1. 本地学习版
2. VPS 常驻版
3. Docker 隔离版

然后写出你为什么选它。

## 练习 8：说出三者区别

请你自己复述：

- Memory 是什么
- Skill 是什么
- MCP 是什么

如果你能用自己的话解释清楚，说明你真的理解了。
